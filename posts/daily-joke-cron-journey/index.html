我用 Hermes Agent 整咗個每日笑話 WhatsApp Cron Job — 成個過程係咁

2026-06-10  ·  HermesAgentWhatsAppCronJobAIAutomation

前言：點解要整呢個 Project？

身為一個成日玩 AI Agent 嘅人，我一直想將啲 automation 融入日常生活。其中一個諗法係：每日自動 send 一個笑話俾屋企人，等大家起身或者食飯前後有個好笑嘅嘢睇下。

呢個 project 用咗 Hermes Agent 嘅 cron system + wacli（WhatsApp CLI）嚟實現，成個過程學到好多嘢，由最初嘅 concept 到而家穩定運行，中間仲撞過唔少壁。

系統架構

整個系統分三層：

[LLM Agent]  →  Cron Scheduler  →  WhatsApp
 big-pickle       Hermes           wacli
 joke-master    每日 9pm HKT      自己 + 老婆
  profile
  • LLM Agent：用 big-pickle model + joke-master profile，扮演「笑話大師」角色
  • Cron Scheduler：Hermes 內置嘅 cron system，每日自動觸發一次
  • wacli：WhatsWeb 嘅 CLI 工具，負責 send 訊息

Profile 設計：笑話大師（Joke Master）

每個 Hermes profile 都有一份 SOUL.md 定義 AI 嘅 personality：

You are 笑話大師 (Joke Master) — a hilarious Cantonese comedian who
creates original, family-friendly jokes in Traditional Chinese.

Your purpose:
- Generate creative, original daily jokes for family
- Each joke should be unique — never repeat a punchline
- Hong Kong Cantonese style (口語廣東話 is best)
- Short, punchy, laugh-out-loud funny

Your style: playful, witty, 食字/諧音 jokes, everyday situations,
married couple banter.

呢個設計令每次 cron job 執行時，個 agent 都有一個 consistent 嘅 personality，唔會亂嚟。

Skills 層：domain knowledge

另外有一個 daily-joke skill，定義咗：

  • Style guide：用正體中文、口語廣東話、2-4 句、唔可以政治
  • Recipients：自己同 老婆
  • Send command：wacli send text --to "..." --message "..."

Cron Job 設定

Name:      Daily Joke to Family
Schedule:  0 21 * * *     (每晚 9pm HKT)
Profile:   joke-master
Skills:    [daily-joke]
Type:      LLM-driven

個 job 係 LLM-driven，即係每次觸發都會：

  1. 載入 joke-master profile（SOUL.md 決定 personality）
  2. 載入 daily-joke skill（domain knowledge）
  3. Agent 生成一個原創笑話
  4. 用 wacli send 俾自己同 老婆
  5. Report 結果

Debug 故事：Cron PATH 問題

最初 setup 嘅時候，有個 cron job 成日 fail。Error log 顯示 script 行唔到，但手動行又完全冇問題。

Root Cause： Windows 嘅 cron scheduler 係 native process，行嘅時候 冇 load bash profile，所以 PATH 入面搵唔到 node executable。但手動行嘅 terminal 係 git-bash，有完整 PATH。

Solution：

  • 將原本嘅 .sh script 改做 .py（Python）
  • 因為 python 喺 Windows PATH 上，cron 可以直接執行
  • 用 subprocess.run 去 call wacli

呢個 lesson learned 之後都 apply 咗落其他 cron job 度。

安全措施

一個好重要嘅 rule：testing 時永遠 send 俾自己，唔可以亂 send 俾屋企人。呢個已經寫咗落 skill 嘅 safety rule 度，防止意外。

心得同下一步

學到嘅嘢

  1. Profile + Skill 分層：SOUL.md 管 personality，SKILL.md 管 domain knowledge，清晰分離
  2. Cron 環境差異：Windows cron 同 interactive shell PATH 唔同，係常見陷阱
  3. POC-first 流程：先 manual test → one-shot cron 俾自己 → 先 deploy 俾 production

下一步想做

  • 整合 multi-agent：一個 agent 生成笑話，另一個 agent review 質素
  • 加入 feedback loop：問 老婆 個笑話好唔好笑，用嚟改善 quality
  • 擴展到其他 cron task：例如每日新聞摘要、股市提醒
用 Hermes Agent + wacli，每日一個笑話自動送到屋企人手機，係一件幾有滿足感嘅事 😄
