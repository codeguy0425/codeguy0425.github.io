用 Hermes Agent + HyperFrames 自動生成 YouTube Shorts

2026-06-20 · Hermes Agent HyperFrames Shorts
一個 AI Agent 由寫 joke 到出片嘅完整 workflow

前言：點解要用 Agent 整 Shorts

最近開始整 YouTube Shorts，每日一條廣東話笑話片。

最初做法係：寫 script → 人手錄音 → 用剪片軟件拼湊 → export → upload。一條片閒閒地 1-2 個鐘。

做咗幾條之後開始諗成個 workflow，發現好多 steps 都係重複嘅：錄音、排版、export、merge audio。既然係重複，點解唔俾 AI Agent 做？

於是開始學點樣用 Hermes Agent 去 automate 呢啲 manual steps — TTS 自動生成、HTML 自動排版、ffmpeg 自動合成。由一開始「做一條片」變成「教 agent 做一條片」。

結果而家嘅 workflow 係：

諗個 joke → 同 agent 講 → 等幾分鐘 → 出片

成個過程 Agent 幫手做晒 heavy lifting，人只需要做 decision making。

Toolchain

Tool用途
Hermes AgentAgent 框架，串連所有工具
HyperFramesHTML-native 影片 render 引擎，寫 HTML = 出片
edge-ttsMicrosoft Edge TTS，支援廣東話多種 voice
ffmpeg多 track audio 合成、volume 調整、trim

Why HyperFrames？

因為 HyperFrames 用 HTML + GSAP 定義畫面，對 AI Agent 嚟講係 natural format。Agent 寫 HTML 遠比操作 Premiere Pro / DaVinci 容易。Render 出嚟嘅係 deterministic MP4 — 同一 input 永遠出同一 output，啱晒 automation。

工作流（以 Daily Joke Video 為例）

Step 1：Agent 生成對話

User 俾一個 joke concept，例如：

點解天上面啲星星咁遠

Agent 會 rewrite 做 natural 對話，最終 script 例如：

🧒 仔：爸爸，點解天上面啲星星咁遠㗎？
👨 爸爸：因為近嗰啲已經變咗隕石，跌落嚟擲死晒啲人啦。

User approve 先 proceed。

Step 2：TTS 生成

用 edge-tts 生成多角色語音：

edge-tts --voice zh-HK-HiuMaanNeural --text "..." --write-media output.wav
edge-tts --voice zh-HK-WanLungNeural --text "..." --write-media output.wav
edge-tts --voice zh-HK-HiuGaaiNeural --text "..." --write-media output.wav

角色 vs Voice mapping 固定：

角色Voice風格
🧒 仔 / 丙朗HiuMaanNeural女聲，輕柔偏嫩
👨 爸爸 / 教練WanLungNeural男聲，沉穩
📖 旁白HiuGaaiNeural女聲，自然清晰

Pitfall： Edge TTS 太短嘅句子（<15 字）有時會 fail，keep 返每句 20+ 字。

Step 3：HTML Bubble Composition

每句對話用一個 <div class="bubble"> 表示，顏色分角色：

  • 🟤 灰色 = 旁白
  • 🔵 藍色 = 教練 / 爸爸
  • 🟠 桃色 = 仔 / 丙朗
<div id="bubble-n1" class="bubble bubble-narrator clip" data-start="4" data-duration="3" data-track-index="12">
  📖 教練同丙朗講⋯
</div>

一句跟一句模式 — 上一句完 immediately 出下一句，唔 fade，唔 overlap。

Step 4：HyperFrames Render

npm run render

Render 出 silent video（~30-60s, 1-2MB），之後再 merge audio。

Step 5：ffmpeg Audio Merge

將多個 TTS + bgm + sfx 用 adelay 同 amix 合成：

ffmpeg -i video.mp4 \
  -i tts-1.wav -i tts-2.wav ... \
  -i bgm.mp3 -i laugh.wav -i credit.wav \
  -filter_complex "
    [1:a]adelay=4000|4000,apad=whole_dur=33[t1];
    ...
    [8:a]volume=1.8,atrim=end=28.5,apad=whole_dur=33[bgm];
    ...
    [t1]...[bgm]...[credit]amix=inputs=10:duration=first[audio]
  " \
  -map 0:v -map "[audio]" final.mp4

Total process：~5-10 分鐘一條片。

可重用框架

每一集嘅 template 係一樣嘅，只需要改 3 樣嘢：

要改嘅位置
🗣️ TTS filessrc/audio/tts/tts-epN-*.wav
📝 Bubble textindex.html 嘅 bubble divs
⏱️ TimingsHTML data-start/data-duration + ffmpeg delay

Template Structure：

daily-joke-video/
├── index.html              ← 主 composition（改 bubble text + timing）
├── src/audio/
│   ├── tts/                ← TTS files（換 joke 時 regenerate）
│   ├── bgm/                ← BGM（重用）
│   └── sfx/                ← Sound effects（重用）
└── renders/
    └── epN-dual-voice-final.mp4  ← Output

每一集新 joke 嘅 workflow：

  1. User approve script
  2. Agent regenerate TTS（3 個 voice x 3-6 句）
  3. Agent copy template index.html，改 bubble text + timing
  4. Agent render + merge
  5. User review

即係話： 第時有新 joke，我只需要一句：

用呢個 joke 做 EP4

Agent 就自動行晒成個 pipeline。

人機協作流程

邊啲位 Agent 做，邊啲位人做：

🧑 User 俾 joke concept
    │
    ▼
🤖 Agent rewrite 做對話腳本
    │
    ▼
🧑 User approve / 改 script
    │
    ▼
🤖 Agent generate TTS + HTML + render + merge
    │
    ▼
🧑 User review final video
    │
    🔄 如果唔滿意 → 改一句話 → Agent 重新 generate + rerender

Agent 做嘅：

  • TTS 生成（3 voice x 6 files）
  • HTML bubble composition
  • GSAP timeline 計算 timing
  • HyperFrames render
  • ffmpeg 多 track audio merge
  • 重複 rerender 直到滿意

人做嘅：

  • Joke 構思同方向
  • Script 審批（地獄唔地獄？夠唔夠好笑？）
  • Timing 調整指示
  • Final output 驗收

Key insight： 人負責 creativity + decision making，Agent 負責 execution + iteration。呢個分工最有效率。

角色聲線配搭

三把廣東話 voice 嘅揀選原因：

Voice角色揀選原因
HiuMaanNeural 🧒仔 / 丙朗女聲偏柔，扮細路自然，同爸爸聲有明顯對比
WanLungNeural 👨爸爸 / 教練男聲沉穩，適合「爸爸式」地獄答案
HiuGaaiNeural 📖旁白女聲清晰中性，適合敘事，同 HiuMaan 有分別唔會混淆

聲線配搭嘅原則：

  • 每個角色要 auditorily distinct — 觀眾一聽就知邊個講嘢
  • 父子對話用 gender contrast（男女聲），自然分到
  • 旁白用第三把聲，同主角色分開

Edge TTS 雖然唔係最自然（對比 ElevenLabs / OpenAI TTS），但勝在免費、速度快、batch generate 冇 quota 限制，啱晒 rapid iteration 嘅 workflow。

AI Agent x 內容創作

Agent 大幅降低咗影片製作嘅門檻：

  • 傳統剪片： 錄音 → 剪 audio → 排版 → 加動畫 → export → 1-2 小時
  • Agent workflow： 諗 joke → approve → agent 做晒 → 5-10 分鐘

以往拍片要識剪片軟件、要識 timing、要識 export settings。而家只需要識 「同 agent 講想做咩」。

AI Agent 唔係取代創作者，而係將 execution time 由 hour 級降到 minute 級，令人可以專注喺創意本身。

最好嘅 content tool 係你唔覺得自己用緊 tool。Agent 就係呢個 direction。
