前言：點解要整呢個 Project？

世界盃2026就嚟開波（其實已經開咗），作為一個球迷＋AI/automation 愛好者，自然想整返個 website 嚟 track 賽果、積分榜、淘汰賽走勢。與其用坊間嘅 app，不如自己起一個仲可以玩下新 tech。

今次我決定用 OpenCode（一個 AI-assisted coding CLI tool）嚟開發，成個 project 由零開始，主要想做幾樣嘢：

  1. 一個 mobile-friendly 嘅 web app，香港時間顯示
  2. 自動更新比數，唔使 manual update
  3. 中英雙語，方便自己同朋友睇
  4. 整合 ViuTV 免費直播時間表

結果用咗大概一個 weekend 就搞掂咗，而家個 site 行緊喺 GitHub Pages，每日自動更新賽果。

數據從邊度嚟？openfootball

最關鍵嘅問題係：世界盃數據去邊度搵？

好彩有人整咗個 openfootball/worldcup.json — 一個 open-source 嘅足球數據庫，以 JSON 格式提供世界盃賽程同賽果。

用法超簡單：

const OPENFOOTBALL_URL = 'https://raw.githubusercontent.com/openfootball/worldcup.json/master/2026/worldcup.json'

const res = await fetch(OPENFOOTBALL_URL)
const data = await res.json()

呢個 JSON 包含咗全部 104 場賽事嘅：

  • 日期、時間
  • 對賽隊伍
  • 賽果（半場／全場）
  • 入球球員同分鐘數
  • 場地

我嘅策略係：將 openfootball 嘅 data 轉換成 TypeScript source code，直接 import 入個 app 度。咁樣做有好處：

  • 完全 type-safe
  • 唔使 runtime API call（無 loading state）
  • 可以 static deploy 去 GitHub Pages

個 update-scores.mjs script 每隔一段時間就 fetch 一次 openfootball，parse 最新賽果，然後喺 CI build 嘅時候自動 update src/data/matches.ts。

最大嘅挑戰：Group Standings 要自己計

呢個係成個 project 最有趣嘅部分。

openfootball 嘅 JSON 雖然有齊賽果，但係佢 無提供 group standings（積分榜）。即係話，佢唔會俾你「Group A 嘅排名係 Mexico 6分、Korea 3分...」呢類 data。

所以我必須自己寫一個 computeStandings() function：

export function computeStandings(group: string, matches: Match[]): GroupStanding[] {
  const teamIds = new Set<string>()
  const groupMatches = matches.filter(m =>
    m.stage === 'group' && m.group === group && m.score1 !== undefined
  )

  // Collect all teams in this group
  for (const m of groupMatches) {
    teamIds.add(m.team1Id)
    teamIds.add(m.team2Id)
  }

  // Initialise standings
  const standings = new Map<string, GroupStanding>()
  for (const id of teamIds) {
    standings.set(id, {
      team: id, played: 0, won: 0, drawn: 0, lost: 0,
      gf: 0, ga: 0, gd: 0, pts: 0, form: []
    })
  }

  // Compute from match results
  for (const m of groupMatches) {
    const s1 = standings.get(m.team1Id)!
    const s2 = standings.get(m.team2Id)!
    s1.played++; s2.played++
    s1.gf += m.score1!; s1.ga += m.score2!
    s2.gf += m.score2!; s2.ga += m.score1!

    if (m.score1 > m.score2) {
      s1.pts += 3; s1.form.push('W'); s2.form.push('L')
    } else if (m.score1 < m.score2) {
      s2.pts += 3; s2.form.push('W'); s1.form.push('L')
    } else {
      s1.pts += 1; s2.pts += 1
      s1.form.push('D'); s2.form.push('D')
    }
  }

  // Sort: points → GD → GF → alphabetical
  return Array.from(standings.values())
    .map(s => ({ ...s, gd: s.gf - s.ga }))
    .sort((a, b) => {
      if (b.pts !== a.pts) return b.pts - a.pts
      if (b.gd !== a.gd) return b.gd - a.gd
      if (b.gf !== a.gf) return b.gf - a.gf
      return a.team.localeCompare(b.team)
    })
}

成個邏輯其實唔複雜：

  1. 搵晒某組嘅所有已完場比賽
  2. 初始化每隊嘅記錄（賽、勝、和、負、入球、失球）
  3. 逐場計分（贏3分、和1分、輸0分）
  4. Sort：先比分數，再比得失球差，再比入球數

好玩在於： 因為 openfootball 無提供 standings，迫住你要自己寫 business logic，反而令個 codebase 更完整，亦都更理解 FIFA 嘅計分規則。

仲有一個小問題：每組有4隊，如果某隊仲未踢過任何比賽，openfootball 嘅 data 唔會包括佢。所以我需要確保 standings table 永遠顯示晒4隊，就算未開波都要 show 0分：

// Fill in teams that haven't played yet
for (const tid of groupInfo.teams) {
  if (!computedIds.has(team?.name || tid)) {
    computed.push({
      team: team?.name || tid,
      played: 0, won: 0, drawn: 0, lost: 0,
      gf: 0, ga: 0, gd: 0, pts: 0, form: []
    })
  }
}

自動更新比分嘅 CI Pipeline

世界盃係進行中嘅賽事，每日都有新賽果。如果每次都要手動更新就太麻煩啦。

所以我 set 咗個 GitHub Actions CI/CD pipeline，每次 push 去 master 嘅時候自動做三件事：

steps:
  - uses: actions/checkout@v4
  - uses: actions/setup-node@v4
  - run: npm ci
  - run: npm run update-scores    # 1️⃣ Fetch latest scores
  - run: node scripts/gen-sitemap.mjs  # 2️⃣ Regenerate sitemap
  - run: npm run build            # 3️⃣ Build & deploy

重點係 npm run update-scores — 呢個 script 會：

  1. Fetch openfootball JSON
  2. 逐場比賽 match 返 src/data/matches.ts 嘅 entry（by match number 或者 by team pair）
  3. 如果有新賽果，就 update score1 / score2 同埋入球資料
  4. 寫返落個 TypeScript file
// Simplified version of update-scores.mjs
for (let i = 0; i < lines.length; i++) {
  const idMatch = lines[i].match(/ id: (\d+),/)
  if (!idMatch) continue

  const ofm = byNum.get(parseInt(idMatch[1]))
  if (!ofm || !ofm.score?.ft) continue

  // Update score
  lines[i] = lines[i].replace(
    /score1: \d+, score2: \d+/g,
    `score1: ${ofm.score.ft[0]}, score2: ${ofm.score.ft[1]}`
  )

  // Update goal details
  if ('goals1' in ofm || 'goals2' in ofm) {
    const goalsStr = buildGoalsString(ofm, team1Id, team2Id)
    lines[i] = lines[i].replace(/goals: \[[^\]]*\]/, goalsStr)
  }
}

每次 CI build 嘅時候都 fetch 最新賽果，變相做到 每日自動更新。仲有 workflow_dispatch 可以手動 trigger。

總結同心得

成個 project 最深刻嘅體會有幾點：

  1. OpenCode + AI-assisted coding 真係好快 — 由 concept 到 deploy 唔使一個 weekend，尤其係 boilerplate 同 UI 嘅 generation 速度驚人
  2. openfootball 係個好 source，但 data 唔 complete — 有賽果但無 standings，迫你寫 business logic 反而學到更多
  3. Static data + CI update 係個好 pattern — 唔需要 backend server，唔需要 database，靠 CI 每日更新 static files 已經做到 dynamic 效果
  4. GitHub Actions free tier 已經夠做 — 每日自動 fetch + build + deploy，完全零成本
